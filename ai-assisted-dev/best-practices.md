## 概述

### 什么是 AI 辅助开发

AI 辅助开发是指使用 AI 工具（如 Claude Code、GitHub Copilot）来辅助软件开发过程，包括：
- 代码生成和编辑
- 问题诊断和调试
- 文档编写和整理
- 代码审查和优化
- 学习新技术

### 核心优势

- ⚡ **提高效率**：自动化重复性工作
- 🎯 **减少错误**：遵循最佳实践
- 📚 **加速学习**：即时获取知识
- 🔄 **快速迭代**：快速验证想法
- 📝 **改善文档**：自动生成文档

### 适用场景

**最适合**：
- ✅ 项目初始化和脚手架搭建
- ✅ 重复性代码编写
- ✅ 文档编写和整理
- ✅ 问题排查和调试
- ✅ 代码重构和优化
- ✅ 学习新技术和框架

**需谨慎**：
- ⚠️ 复杂的业务逻辑设计
- ⚠️ 安全敏感的代码
- ⚠️ 性能关键的核心算法
- ⚠️ 需要深度领域知识的场景

## 沟通技巧

### 清晰的任务描述

**✅ 好的示例**：
```
我需要创建一个 Python 脚本来处理 CSV 文件：
1. 读取 data.csv 文件
2. 过滤出 age > 18 的行
3. 按 name 列排序
4. 输出到 output.csv

请使用 pandas 库，添加错误处理。
```

**❌ 不好的示例**：
```
帮我写个处理数据的脚本
```

### 提供充足的上下文

**包含关键信息**：
- 项目背景和目标
- 技术栈和依赖
- 现有代码结构
- 约束条件（性能、安全等）
- 预期输出

**示例**：
```
我正在开发一个 ROS2 节点，用于处理激光雷达数据。
- 使用 Python3
- ROS2 Humble 版本
- 需要订阅 /scan 话题
- 发布处理后的数据到 /filtered_scan
- 需要过滤掉距离 < 0.1m 的点
```

### 分步骤进行

**复杂任务拆分**：
```
第一步：先帮我设计整体架构
第二步：实现核心功能模块
第三步：添加错误处理
第四步：编写单元测试
第五步：完善文档
```

**逐步验证**：
- 每完成一步验证结果
- 确认无误后继续下一步
- 避免一次性完成所有工作

### 及时反馈

**提供反馈**：
```
✅ "这个方案很好，请继续"
✅ "这里有个问题：..., 请修改"
✅ "我更倾向于使用 X 方案而不是 Y"
❌ 不提供反馈，让 AI 猜测你的想法
```

## 任务规划

### 使用任务清单

**创建待办清单**：
```markdown
- [ ] 创建项目目录结构
- [ ] 初始化 Git 仓库
- [ ] 设置虚拟环境
- [ ] 安装依赖包
- [ ] 编写核心功能
- [ ] 添加单元测试
- [ ] 编写 README
```

**跟踪进度**：
- 使用 TodoWrite 工具（Claude Code）
- 标记任务状态（pending/in_progress/completed）
- 及时更新进度

### 优先级管理

**重要且紧急**：
1. 修复关键 bug
2. 解决阻塞问题

**重要不紧急**：
1. 代码重构
2. 文档完善
3. 性能优化

**不重要但紧急**：
1. 小的 bug 修复
2. 格式调整

**不重要不紧急**：
1. 代码美化
2. 注释优化

### 时间估算

**合理估算**：
- 简单任务：10-30 分钟
- 中等任务：1-3 小时
- 复杂任务：半天到 1 天
- 超大任务：拆分成小任务

## 代码审查

### AI 生成代码的审查要点

**安全性检查**：
- [ ] 是否有 SQL 注入风险
- [ ] 是否有 XSS 漏洞
- [ ] 是否正确处理用户输入
- [ ] 是否有敏感信息泄露
- [ ] 是否使用了安全的加密方法

**功能正确性**：
- [ ] 逻辑是否正确
- [ ] 边界条件是否处理
- [ ] 错误处理是否完善
- [ ] 是否符合需求

**代码质量**：
- [ ] 命名是否清晰
- [ ] 结构是否合理
- [ ] 是否遵循编码规范
- [ ] 是否有重复代码
- [ ] 注释是否充分

**性能考虑**：
- [ ] 算法复杂度是否合理
- [ ] 是否有性能瓶颈
- [ ] 内存使用是否合理
- [ ] 是否有资源泄露

### 测试策略

**单元测试**：
```python
# 要求 AI 生成测试
def test_filter_data():
    input_data = [1, 2, 3, 4, 5]
    result = filter_data(input_data, threshold=3)
    assert result == [4, 5]
```

**集成测试**：
- 测试模块间的交互
- 验证端到端流程

**边界测试**：
- 空输入
- 极大/极小值
- 特殊字符
- 并发场景

## 知识管理

### 即时记录

**学习过程记录**：
```markdown
## 学习 Docker 网络

### 问题
如何让容器间互相通信？

### AI 给的方案
1. 使用 bridge 网络
2. 使用 docker-compose 的网络配置

### 实践结果
方案 2 更简单，在 docker-compose.yml 中配置：
\`\`\`yaml
networks:
  app-network:
    driver: bridge
\`\`\`

### 经验总结
- docker-compose 自动创建网络
- 容器可以通过服务名互相访问
- 生产环境建议自定义网络名称
```

### 问题解决方案库

**建立问题库**：
```markdown
# troubleshooting/docker-issues.md

## 容器无法启动

### 错误信息
```
Error: Port 8080 is already in use
```

### 原因
端口被占用

### 解决方案
\`\`\`bash
# 查找占用端口的进程
lsof -i :8080

# 杀死进程
kill -9 <PID>

# 或者修改 docker-compose.yml 使用其他端口
ports:
  - "8081:8080"
\`\`\`
```

## 常见陷阱

### 过度依赖

**问题**：
- 不思考直接使用 AI 代码
- 不理解代码就提交
- 完全依赖 AI 调试

**解决**：
- ✅ 理解每一行代码
- ✅ 自己先尝试解决
- ✅ 把 AI 当助手不是替代

### 安全风险

**问题**：
- AI 生成的代码可能有漏洞
- 可能使用过时的安全实践
- 可能暴露敏感信息

**解决**：
- ✅ 安全敏感代码人工审查
- ✅ 使用安全扫描工具
- ✅ 遵循安全最佳实践
- ✅ 不要在提示中包含敏感信息

### 质量问题

**问题**：
- 代码不符合项目规范
- 没有错误处理
- 缺少文档和注释

**解决**：
- ✅ 明确告知编码规范
- ✅ 要求添加错误处理
- ✅ 要求添加文档注释
- ✅ 代码审查和测试

### 上下文丢失

**问题**：
- 对话太长 AI 忘记前文
- 任务切换导致混乱

**解决**：
- ✅ 及时总结关键信息
- ✅ 新任务开新会话
- ✅ 提供必要的上下文
- ✅ 使用知识库记录

## 最佳实践

### SOP：AI 辅助开发标准流程

#### ✅ 开始新任务

**1. 明确需求**：
```
- [ ] 理解任务目标
- [ ] 确定技术栈
- [ ] 明确约束条件
- [ ] 准备必要资源
```

**2. 与 AI 沟通**：
```
提供完整上下文：
- 任务目标和背景
- 技术栈和依赖
- 现有代码结构
- 特殊要求
```

**3. 制定计划**：
```
请 AI 帮助：
- 分析任务复杂度
- 拆分为子任务
- 创建任务清单
- 估算时间
```

**4. 执行开发**：
```
- [ ] 按步骤进行
- [ ] 每步验证结果
- [ ] 及时反馈调整
- [ ] 更新任务状态
```

**5. 代码审查**：
```
- [ ] 功能是否正确
- [ ] 是否有安全问题
- [ ] 代码质量如何
- [ ] 是否有测试
```

**6. 测试验证**：
```
- [ ] 运行单元测试
- [ ] 手动测试关键功能
- [ ] 测试边界情况
- [ ] 性能测试（如需要）
```

**7. 文档整理**：
```
- [ ] 更新代码注释
- [ ] 更新 README
- [ ] 记录关键决策
- [ ] 更新知识库
```

**8. 提交代码**：
```
- [ ] 审查所有更改
- [ ] 编写清晰的提交信息
- [ ] 推送到远程仓库
```

#### ✅ 问题排查流程

**1. 描述问题**：
```
向 AI 提供：
- 错误信息（完整的）
- 复现步骤
- 环境信息
- 相关代码
```

**2. 分析原因**：
```
请 AI 帮助：
- 分析错误信息
- 定位问题代码
- 解释可能原因
```

**3. 尝试解决**：
```
- [ ] 实施建议方案
- [ ] 验证是否解决
- [ ] 如未解决，提供新信息
- [ ] 继续迭代
```

**4. 总结记录**：
```
- [ ] 记录问题和解决方案
- [ ] 更新到知识库
- [ ] 分享给团队
```

#### ✅ 学习新技术流程

**1. 设定目标**：
```
明确学习目标：
- 我想学什么
- 达到什么水平
- 应用到什么场景
```

**2. 制定计划**：
```
请 AI 帮助：
- 推荐学习路径
- 列出关键知识点
- 提供学习资源
```

**3. 动手实践**：
```
- [ ] 跟随教程练习
- [ ] 完成小项目
- [ ] 向 AI 提问
- [ ] 记录笔记
```

**4. 知识整理**：
```
- [ ] 创建知识库文档
- [ ] 总结关键概念
- [ ] 记录最佳实践
- [ ] 整理常见问题
```
