### 必备要素

**每个文档应包含**：

1. **标题和简介**：清晰说明文档主题
2. **目录**：方便快速导航
3. **概述**：背景和应用场景
4. **详细内容**：技术细节和示例
5. **SOP/检查清单**：标准操作流程
6. **常见问题**：问题和解决方案
7. **最佳实践**：经验总结
8. **学习资源**：参考链接
9. **学习记录**：时间和要点

### Markdown 格式规范

**标题层级**：
```markdown
# H1：文档标题（每个文档只有一个）
## H2：主要章节
### H3：子主题
#### H4：详细内容
```

**代码块**：
````markdown
```bash
# Bash 命令
git status
```

```python
# Python 代码
def hello():
    print("Hello")
```
````

**列表**：
```markdown
# 无序列表
- 项目 1
- 项目 2
  - 子项目 2.1

# 有序列表
1. 步骤 1
2. 步骤 2

# 任务列表
- [ ] 未完成任务
- [x] 已完成任务
```

**强调**：
```markdown
**加粗**：重要内容
*斜体*：强调

> 引用：重要说明
```

**符号**：

## 知识分类体系

### 技术领域分类

#### 1. 开发工具
- **Git/GitHub**：版本控制、协作开发
- **AI 辅助开发**：Claude Code、Copilot
- **编辑器**：VSCode、Vim
- **容器化**：Docker、Kubernetes

#### 2. 编程语言
- **Python**：语法、标准库、框架
- **C++**：现代 C++、STL、性能优化
- **JavaScript/TypeScript**：前端开发
- **Shell**：脚本编写、自动化

#### 3. 框架和库
- **Web 框架**：React、Vue、Django
- **机器学习**：PyTorch、TensorFlow
- **机器人**：ROS、ROS2
- **数据库**：MySQL、PostgreSQL、MongoDB

#### 4. 系统和架构
- **操作系统**：Linux、进程管理
- **网络**：协议、API 设计
- **分布式系统**：微服务、消息队列
- **云服务**：AWS、Azure、GCP

#### 5. 专业领域
- **计算机视觉**：OpenCV、YOLO
- **自然语言处理**：Transformer、LLM
- **机器人技术**：导航、控制、感知
- **嵌入式开发**：单片机、RTOS

### 文档类型分类

#### 1. 教程型（Tutorial）
- **特点**：从零开始，循序渐进
- **适用**：新手入门，系统学习
- **命名**：`getting-started.md`、`tutorial.md`

#### 2. 指南型（Guide）
- **特点**：解决特定问题
- **适用**：完成具体任务
- **命名**：`workflow.md`、`best-practices.md`

#### 3. 参考型（Reference）
- **特点**：速查表、API 文档
- **适用**：快速查找信息
- **命名**：`commands-reference.md`、`api-reference.md`

#### 4. 解释型（Explanation）
- **特点**：深入讲解原理
- **适用**：理解底层机制
- **命名**：`how-it-works.md`、`architecture.md`

#### 5. SOP 型（Standard Operating Procedure）
- **特点**：标准化流程
- **适用**：重复性操作
- **命名**：`sop.md`、`checklist.md`

## 更新维护流程

### 创建新文档

**步骤 1：确定分类**
```bash
# 确定文档应该放在哪个目录
# 如果目录不存在，创建新目录
mkdir -p KnowledgeBase/category-name
```

**步骤 2：创建文档**
```bash
# 使用文档模板创建新文件
touch KnowledgeBase/category-name/topic-name.md
```

**步骤 3：编写内容**
- 遵循文档结构模板
- 包含所有必备要素
- 添加实际示例

**步骤 4：提交到 Git**
```bash
cd KnowledgeBase
git add .
git status  # 确认要提交的文件
git commit -m "docs: 添加 [主题] 文档"
git push
```

### 更新现有文档

**步骤 1：定位文档**
```bash
# 使用 find 或 grep 查找
find KnowledgeBase -name "*keyword*"
grep -r "关键词" KnowledgeBase/
```

**步骤 2：编辑内容**
```bash
# 使用编辑器打开
vim KnowledgeBase/path/to/file.md
```

**步骤 3：更新学习记录**
```markdown
## 学习记录
- 📅 2025-11-05：创建文档
- 📅 2025-11-06：更新 XXX 部分，新增 YYY 示例
```

**步骤 4：提交更改**
```bash
git add KnowledgeBase/path/to/file.md
git commit -m "docs: 更新 [主题] - 新增 [内容]"
git push
```

## 最佳实践

### 文档编写

**1. 以读者为中心**
- ✅ 明确目标读者（初学者/中级/高级）
- ✅ 使用清晰简洁的语言
- ✅ 提供充足的上下文
- ❌ 不要假设读者知识背景

**2. 示例驱动**
- ✅ 提供可运行的代码示例
- ✅ 展示实际应用场景
- ✅ 包含输出结果
- ❌ 不要只有理论没有实践

**3. 结构化组织**
- ✅ 使用清晰的层级结构
- ✅ 相关内容归类整理
- ✅ 提供目录和导航
- ❌ 不要内容杂乱无章

**4. 持续更新**
- ✅ 及时记录新知识
- ✅ 更新过时内容
- ✅ 补充实践经验
- ❌ 不要一写了之不再维护

### 知识管理

**1. 即时记录**
```bash
# 学习时立即记录
# 解决问题后立即总结
# 不要等到忘记才记录
```

**2. 结构化思维**
```markdown
# 按照统一的结构组织
1. 是什么（What）
2. 为什么（Why）
3. 怎么做（How）
4. 注意事项（Caution）
5. 实践经验（Practice）
```

**3. 关联引用**
```markdown
# 在文档中建立链接
- 相关主题：[Git 工作流程](../git-github/workflow.md)
- 参考文档：[Claude Code 工作流](../ai-assisted-dev/claude-code-workflow.md)
```

**4. 定期复习**
- 每周复习本周学习内容
- 每月总结学习收获
- 通过实践巩固知识

### Git 工作流

**1. 语义化提交**
```bash
# 新增文档
git commit -m "docs: 添加 [主题] 文档"

# 更新文档
git commit -m "docs: 更新 [主题] - 新增 [内容]"

# 修正文档
git commit -m "docs: 修正 [主题] 中的错误"

# 重构文档
git commit -m "docs: 重构 [主题] 文档结构"
```

**2. 小步提交**
- ✅ 完成一个主题就提交
- ✅ 每次提交聚焦一个改动
- ❌ 不要积累大量改动一次提交

**3. 清晰的提交历史**
```bash
# 查看提交历史
git log --oneline

# 应该能清楚看到知识库的演进
- docs: 添加 Git 工作流程文档
- docs: 添加 Claude Code 工作流文档
- docs: 更新 Git 工作流 - 新增分支管理
```

## 工作流程示例

### 场景 1：学习新技术

**步骤**：
1. 确定技术分类
2. 创建文档目录
3. 边学习边记录
4. 整理成文档
5. 提交到知识库

**示例**：
```bash
# 1. 学习 Docker
mkdir -p KnowledgeBase/tools/docker

# 2. 创建文档
touch KnowledgeBase/tools/docker/getting-started.md

# 3. 边学边记录
# 记录命令、概念、问题、解决方案

# 4. 整理文档
# 按照模板组织内容

# 5. 提交
cd KnowledgeBase
git add tools/docker/
git commit -m "docs: 添加 Docker 入门文档"
git push
```

### 场景 2：解决问题后总结

**步骤**：
1. 定位相关文档
2. 在"常见问题"章节添加
3. 记录问题、原因、解决方案
4. 提交更新

**示例**：
```bash
# 1. 找到相关文档
# 例如：git-github/workflow.md

# 2. 编辑文档
vim KnowledgeBase/git-github/workflow.md

# 3. 添加问题和解决方案
## 常见问题

### 问题 X：描述
**原因**：...
**解决**：...

# 4. 提交
git add git-github/workflow.md
git commit -m "docs: 更新 Git 工作流 - 新增问题解决方案"
git push
```

### 场景 3：项目结束后总结 SOP

**步骤**：
1. 回顾项目流程
2. 提取标准化步骤
3. 创建 SOP 文档
4. 包含检查清单
5. 提交到知识库

**示例**：
```markdown
# 项目部署 SOP

## ✅ 部署前检查清单
- [ ] 代码已合并到主分支
- [ ] 所有测试通过

2. 拉取最新代码
3. 安装依赖

5. 重启服务
6. 验证部署

## ✅ 部署后验证
```
